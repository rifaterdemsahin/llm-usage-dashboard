Why Claude & Gemini don't auto-pull

The real errors, and what's needed to see usage
📊 Dashboard 🔎 Manual checks ⚖️ Compare ⚠️ Errors

OpenRouter works because it exposes a key-scoped usage endpoint (GET /api/v1/key) that returns spend with the same key used to call models. Claude and Gemini don't. The API keys we hold can call the models but cannot read usage — that data sits behind different credentials. And critically, the chat subscriptions (Claude Pro, Gemini Advanced) expose no usage API at all.

Claude (Anthropic) USAGE BLOCKED

The stored key is valid — it returns 200 from the Messages API — but the Admin Usage/Cost API rejects it.

What actually happens

# Messages API with our key — works
POST /v1/messages                       → HTTP 200

# Admin Usage Report with the same key — rejected
GET /v1/organizations/usage_report/messages
→ HTTP 401
{"type":"error","error":{"type":"authentication_error",
 "message":"invalid x-api-key"}}

Why

  • Usage & cost are only on the Admin API (/v1/organizations/usage_report/*, /cost_report).
  • It needs an Admin API key (sk-ant-admin…), which only an Organization Owner can create — a normal sk-ant-api… key can't read it.
  • Browsers can't call it directly (CORS) — must be server-side, which we already do for OpenRouter.

What needs to happen to see it

  • An org owner creates an Admin key in Console → Settings → Admin keys.
  • Store it as a secret (e.g. ANTHROPIC_ADMIN_KEY); the backend then calls the usage + cost reports and fills the Claude card live.
Subscriptions ≠ API. A Claude Pro / Max plan ($20 / $100) is the claude.ai chat product. Its usage is not exposed by any API and can't appear here — only API (Console) billing can. To track Claude here you must use the API platform with an Admin key, not the chat subscription.
Gemini (Google) USAGE BLOCKED

The AI Studio key can list/call models, but there is no usage or cost endpoint on it at all.

What actually happens

# List models with our key — works
GET /v1beta/models?key=…                → HTTP 200

# Usage / cost for that key
→ no such endpoint exists.
  Usage & spend live in Google Cloud, not the Gemini API.

Why

  • The Generative Language (AI Studio) API only serves inference — it reports no token/cost totals.
  • Usage & cost come from Google Cloud: the Cloud Billing API (cost) and Cloud Monitoring API (request/token metrics).
  • Those require a Google Cloud project with billing enabled and a service account with the right roles — not a bare API key.

What needs to happen to see it

  • Use a GCP service account JSON ✓ we have gcp-service-account-json in the vault with roles/monitoring.viewer and roles/billing.viewer.
  • Enable the Cloud Billing and Cloud Monitoring APIs on the project that owns the Gemini key.
  • The backend then reads token metrics (generativelanguage.googleapis.com/…) and cost from billing, and fills the Gemini card.
Subscriptions ≠ API. Gemini Advanced (Google One AI) is the consumer chat plan and has no API access whatsoever. Only pay-as-you-go API usage tied to a Cloud project can be measured here.
OpenRouter LIVE

For contrast — this is why OpenRouter just works:

# Same key that calls models also reports spend
GET /api/v1/key  (Bearer sk-or-…)       → HTTP 200
{"data":{"usage":52.18,"limit":null}}

One key, usage included, callable server-side. That's the bar the other two don't meet.
