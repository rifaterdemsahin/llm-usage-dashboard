Fix connectivity — IP allowlist

Atlas rejects connections from IPs that aren't allowlisted (you'll see remote error: tls: internal error in the Fly logs). Fly's egress IPs are dynamic, so allow from anywhere — the database stays protected by its username/password.

  1. Open Network Access (button above).
  2. Click + ADD IP ADDRESS.
  3. Click ALLOW ACCESS FROM ANYWHERE → fills 0.0.0.0/0.
  4. Add comment fly llm-usage-dashboard → Confirm.
  5. Wait until status shows Active (~1 min), then redeploy / restart the Fly machine.

Data model

One document per user in the llmdash.settings collection. The settings blob is stored as a JSON string to avoid BSON key restrictions; usage results are cached under cache.

// db: llmdash · collection: settings
{
  "_id": "info@pexabo.com",          // the login username
  "data": "{ ...JSON... }",          // keys + per-provider numbers + cache
  "updated": ISODate("…")
}

// decoded "data":
{
  "keys": { "openrouter": "sk-or-…", "claude": "sk-ant-…", "gemini": "AQ.…" },
  "openrouter": { "cost": 52.18, "live": true, "series": [ … ] },
  "claude":     { "cost": 0, "requests": 0, "tokens": 0 },
  "gemini":     { "cost": 0, "requests": 0, "tokens": 0 },
  "cache":      { "openrouter": { "value": {"cost":52.18}, "fetchedAt": "…" } }
}

daily_usage collection

One document per user · date · provider. The first reading of the day sets opening; every later pull updates last. Spent today = last − opening.

// db: llmdash · collection: daily_usage
{
  "_id": "info@pexabo.com|2026-06-22|openrouter",
  "user": "info@pexabo.com",
  "date": "2026-06-22",
  "provider": "openrouter",
  "opening": 53.04,                  // cumulative at first reading today
  "last": 53.04,                     // latest cumulative reading
  "updated": ISODate("…")
}

Environment / secrets (Fly)

MONGODB_URIAtlas connection string (mongodb+srv://…), from the vault
MONGODB_DBllmdash — separate DB so the shared cluster's other data is untouched
USAGE_CACHE_TTLseconds before a live usage pull is refreshed (default 3600)

If MONGODB_URI is unset or unreachable, the app falls back to an in-memory store (data resets on restart) — shown as memory in the status above.
